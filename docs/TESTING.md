# GeoXam — Testing Strategy

---

## Принцип

Тестируется только то, что можно тестировать автоматически **без подмены браузерных API**.  
Камера и GPS — это железо. Их тестируют на реальном устройстве руками.

---

## Структура тестов

```
tests/
├── unit/
│   ├── geo/formatter.test.ts     → Node.js, чистые функции, нет браузера
│   └── crypto/crypto.test.ts     → Node.js, SubtleCrypto (Node 20 поддерживает нативно)
│
├── unit/db/
│   └── settings.test.ts          → Реальный Chromium (vitest browser mode)
│                                    Реальный IndexedDB. Никаких fake-indexeddb.
│
└── e2e/
    └── navigation.spec.ts        → Playwright Chromium
                                     Навигация, Gallery CRUD, Settings persistence,
                                     Mask calculator unlock — всё без камеры/GPS.
```

---

## Запуск

```bash
# Юнит тесты чистых функций (Node, мгновенно)
npm test

# DB тесты в реальном Chromium
npm run test:browser

# E2E
npm run build && npm run test:e2e
```

## Что НЕ тестируется автоматически и почему

| Фича | Почему нельзя автоматизировать |
|---|---|
| Съёмка фото | Требует реальный camera stream с устройства |
| GPS координаты | Требует реальный GPS-приёмник |
| Точность GPS | Зависит от окружения (indoor/outdoor) |
| Гироскоп/heading | Датчик устройства |

**Нет никаких `getUserMedia` моков.  
Нет никаких `fake-indexeddb`.  
Нет никаких JS-патчей на `navigator.geolocation`.**

Playwright `context.setGeolocation()` — это не мок. Это штатный Chromium DevTools Protocol, который подменяет координаты на уровне браузерного процесса (тот же механизм, что в DevTools → Sensors). Используется ТОЛЬКО для тестирования UI-поведения при изменении статуса GPS, не для тестирования самого GPS.

## Ручное тестирование на устройстве

### Подготовка
1. `npm run build`
2. Загрузить `dist/` на HTTPS хост (GitHub Pages или `ngrok`)
3. Открыть на реальном Android/iOS устройстве

### Чек-лист ручного тестирования

#### Камера
- [ ] Видоискатель запускается, нет задержки
- [ ] Crosshair отображается поверх видео
- [ ] Кнопка переключения камеры (front/back) работает
- [ ] Съёмка срабатывает на нажатие кнопки
- [ ] Съёмка срабатывает на кнопку громкости (если поддерживается)
- [ ] После съёмки — flash успеха, видоискатель возобновляется
- [ ] Разные разрешения (HD/FHD/4K/Max) сохраняют корректный размер файла

#### GPS
- [ ] Badge показывает «Acquiring…» при запуске
- [ ] Badge переходит в зелёный после получения фикса
- [ ] Badge показывает точность (±N m)
- [ ] При отказе в разрешении — badge «GPS denied», съёмка работает (coords = null)
- [ ] Overlay на фото содержит реальные координаты, совпадающие с местом съёмки
- [ ] DMS формат парсится корректно (проверить в Google Maps)

#### Галерея
- [ ] Фото появляется сразу после съёмки
- [ ] Thumbnail отображается корректно
- [ ] Просмотр полного фото
- [ ] Download сохраняет файл с координатами в overlay
- [ ] Удаление фото
- [ ] Создание папки, перемещение фото

#### Маски
- [ ] Калькулятор: работает как нормальный калькулятор
- [ ] Калькулятор: последовательность `1337=` открывает приложение
- [ ] Календарь: тройной тап на сегодня открывает приложение
- [ ] Блокнот: `:::` + тап угла открывает приложение
- [ ] PIN: 3 неверных попытки → 30с блокировка
- [ ] Авто-блокировка: свернуть на 5+ минут → при возврате показывается маска

#### PWA
- [ ] Устанавливается на домашний экран (Android: баннер, iOS: Share → Add to Home Screen)
- [ ] Работает без интернета после установки
- [ ] После обновления — toast «Update available»
